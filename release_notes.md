v2.0.0 - 适配即梦 CLI v1.4.2 & 任务管理功能

> **本次更新的核心驱动：适配官方即梦 CLI v1.4.2**。
> 新版 CLI 将 submit_id 格式从旧版 16 位十六进制（如 `fca1295b3a90955e`）升级为标准 UUID（如 `0d939fcb-ff29-4b59-b992-ddcf73bd2c3b`），旧版本项目无法正确解析新格式 ID，导致任务轮询失效。本次更新完整适配新格式，同时新增多项管理功能。
>
> ⚠️ **最低 CLI 版本要求：v1.4.2**。如你仍在使用旧版 CLI，请先升级再使用本版本 API。

## 🔄 更新内容 / Changelog

- 🔧 **[核心] 适配 CLI v1.4.2 新版 submit_id 格式**：`extractSubmitInfo` 优先解析 JSON 中的 UUID 格式 submit_id，兼容旧版 16 位 hex 格式作为 fallback。
- 🔧 **[核心] 同步提取并持久化 logid**：新版 CLI 返回 Bytedance 请求追踪 ID（logid），现已在提交时一并保存至数据库，便于问题排查与客诉处理。
- 🗄️ **数据库新增 `jimengLogId` 字段**：升级包含自动迁移脚本，无需手动操作。
- 🆕 **管理后台新增「任务管理」分页**：查看所有任务状态，支持按状态筛选与分页。
- 🆕 **管理员手动强制失败**：一键将卡住的任务标记为失败，可附加说明。
- 🆕 **任务重试**：FAILED 任务（有 submit_id）可直接重置为 PROCESSING 重新轮询。
- 🆕 **任务详情弹窗**：完整展示 submit_id、logid、Prompt、结果 URL、失败原因，均可复制。
- 🆕 **24h 超时自动失败**：PROCESSING 超过 24 小时的任务在下次轮询时自动标记 FAILED。

v1.0.1 - 修复 seedance2.0 多模态路由与兼容性提升

## 🔄 更新内容 / Changelog
- 🐛 **Bug修复**：修复了 OpenAI 格式生视频接口中，当仅上传图片（无音视频附件）并指定 seedance2.0 等多模态模型时，因附件类型判定错误导致路由掉入普通多帧/单图模式（默认 3.0 fast）的问题。现在只要指定了多模态模型，将强制且正确地自动路由至多模态通道，极大提升了 API 的模型兼容性与调度准确度。

**1. `jimengCli_api-v2.0.0-windows-x64.zip` (完整部署包 / Full Package)**
