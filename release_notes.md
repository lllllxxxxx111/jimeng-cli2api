v1.0.2 - 任务管理 & 稳定性提升

## 🔄 更新内容 / Changelog

- 🆕 **管理后台新增「任务管理」分页**：可查看所有任务的状态、类型、模型、账号归属，支持按状态筛选与分页浏览。
- 🆕 **管理员手动强制失败**：针对卡住的任务，管理员可在后台一键强制标记为失败（可附加失败原因说明）。
- 🆕 **任务重试**：对于 FAILED 状态且存在 submit_id 的任务，可直接重置为 PROCESSING，由轮询守护程序重新拉取状态。
- 🆕 **任务详情弹窗**：展示完整的 submit_id、logid（火山请求追踪 ID）、Prompt、结果 URL、失败原因等调试信息，均可直接复制。
- 🆕 **24h 超时自动失败**：处于 PROCESSING 状态超过 24 小时的任务，将在下次轮询时自动标记为 FAILED，避免任务永久卡死。
- 🆕 **logid 持久化**：任务提交时同步提取并保存 CLI 返回的 logid（Bytedance 请求追踪 ID），便于排查问题。
- 🗄️ **数据库新增字段**：`Task` 表新增 `jimengLogId` 字段，升级包已包含自动迁移脚本。

v1.0.1 - 修复 seedance2.0 多模态路由与兼容性提升

## 🔄 更新内容 / Changelog
- 🐛 **Bug修复**：修复了 OpenAI 格式生视频接口中，当仅上传图片（无音视频附件）并指定 seedance2.0 等多模态模型时，因附件类型判定错误导致路由掉入普通多帧/单图模式（默认 3.0 fast）的问题。现在只要指定了多模态模型，将强制且正确地自动路由至多模态通道，极大提升了 API 的模型兼容性与调度准确度。

## 📦 该下载哪一个包？(Which package to download?)

**1. `jimengCli_api-v1.0.1-windows-x64.zip` (完整部署包 / Full Package)**
- **使用场景**：适合**第一次使用**或**全新部署**的用户。
- **说明**：包含了完整的 Node 环境依赖、数据库文件映射等。解压后可直接双击 `start.bat` 运行。（注意：你需要按照文档自行下载官方 `dreamina.exe` 放入 `bin` 目录）。

**2. `jimeng-upgrade.zip` (增量升级包 / Upgrade Package)**
- **使用场景**：适合**老用户升级**。
- **说明**：仅包含此次更新的核心程序代码及前端编译文件，**绝对不会覆盖或修改**你现有的 `data` 目录（账号和数据库数据），保证你的运行数据安全。
- **使用方法**：解压该升级包到任意临时目录，双击运行里面的 `upgrade_apply.bat`，输入你老版本的安装目录，脚本会自动覆盖更新并应用可能需要的数据库更新检查。
